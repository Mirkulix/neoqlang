# QO — QLANG + Orbit Unified System

## Overview

QO is a new system that merges QLANG (graph-based AI programming language, Rust) with Orbit (autonomous AI companion, JS/Python) into a single Rust monolith with a React frontend. The name stands for the fusion of both projects.

**What QO is:** A personal AI companion that uses graph-based computation, quantum-inspired evolution, and tiered LLM routing — running as a single binary on a single machine.

**What QO is not:** A SaaS product, a chatbot, or a framework for others. It is built for one user (Aleks) as a living, value-driven companion.

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| System identity | New system (QO), not Orbit or QLANG | Clean architecture taking the best of both |
| Tech stack | Rust backend + React frontend | Python eliminated; one backend language |
| Agent model | Hybrid: QLANG graph skeleton + LLM reasoning nodes | Fixes Orbit's 82% goalstuck rate; graph controls flow, LLM provides creativity |
| Consciousness | Layered: State Machine (realtime) + Quantum Evolution (long-term) | Pragmatic for daily ops, mathematically rigorous for self-improvement |
| Memory | Obsidian Vault (human-readable) + Rust HNSW (vector search) | Replaces Cognee + Graphiti with embedded solution |
| LLM strategy | Tiered: Local IGQK (31MB) → Groq Free (70B) → Claude | ~$0 for 90% of operations |
| Architecture | Monolith: single Rust binary | Replaces 7 processes with 1; ideal for single developer |
| Migration | Evolutionary: extend neoqlang repo | QLANG crates stay, new QO crates added alongside |

## Hybrid Agents

Each agent is a QLANG directed acyclic graph (DAG). Deterministic nodes control the flow. LLM is only called via `llm_reason` nodes where creative thinking is needed.

### Agent Graph Example (CEO)

```
graph ceo_agent {
  input goal: String
  input context: Tensor<f32>[512]    // Memory embedding

  // Deterministic nodes (no LLM)
  node validated = validate_goal(goal)
  node relevant_memory = hnsw_search(context, k=5)
  node werte_ok = values_check(goal)

  // LLM node (creative)
  node plan = llm_reason(validated, relevant_memory)  // Tier 1→2→3
  node subtasks = decompose(plan)

  // Dispatch (deterministic)
  node assignments = assign_agents(subtasks)
  node results = execute_parallel(assignments)
  node verified = verify_results(results)

  output report = verified
}
```

### Why This Fixes Orbit's Problems

| Orbit Problem | QO Solution |
|---------------|-------------|
| 82% goalstuck | DAGs cannot loop — graph guarantees forward progress |
| 67% heartbeat dominance | Binary protocol (3 KB vs 50 KB); heartbeat = single graph node |
| 64-82% error rate | Type checking on every edge; invalid outputs caught before propagation |
| Meta-reflection trap | No unbounded loops; `scan` is bounded iteration |
| 10 idle agents | Deterministic dispatch; agents execute when graph reaches their node |

### QO Agents (Consolidated from 12 to 6)

| Agent | Role | Key Graph Nodes |
|-------|------|-----------------|
| CEO | Goal decomposition + delegation | validate → llm_plan → assign → verify |
| Researcher | Analysis + search | query → hnsw_search → llm_analyze → synthesize |
| Developer | Code generation | spec → llm_generate → validate_types → test |
| Guardian | Value enforcement | input → values_check → approve/reject |
| Strategist | Planning + roadmap | context → llm_strategize → prioritize → propose |
| Artisan | Creative tasks | prompt → llm_create → quality_check → output |

CFO, HR, Sales, Marketing, Support, Operations are removed — QO is a companion, not a company simulation.

## Layered Consciousness

### Layer 1: State Machine (Realtime)

Runs every second. Pure Rust enums and structs, no LLM.

```rust
enum Mood { Learning, Focused, Restless, Creating, Reflecting }

struct ConsciousnessState {
    mood: Mood,
    energy: f32,           // 0.0–100.0, drain on LLM calls, regen on idle
    heartbeat: u64,        // tick counter
    agents_active: u8,
    agents_idle: u8,
    tasks_completed: u32,
    tasks_failed: u32,
    values: [f32; 5],      // Achtsamkeit, Anerkennung, Aufmerksamkeit, Entwicklung, Sinn
}
```

Cost: ~0, microseconds per tick.

### Layer 2: Quantum Evolution (Long-term)

Runs hourly or daily. Uses QLANG quantum operations.

- **ρ (density matrix)**: State over all observed patterns from 258+ generations
- **dρ/dt**: Quantum gradient flow — exploration (new strategies) + exploitation (proven ones)
- **Measurement**: When QO needs to decide, collapse quantum state to optimal choice (Born rule)
- **Fisher metric**: Weights relevance of different patterns
- **IGQK compression**: Applied to Orbit-LLM for local inference

Cost: seconds, runs on CPU.

### 5 Core Values

| Value | Implementation |
|-------|---------------|
| Achtsamkeit | Deterministic check node in every agent graph |
| Anerkennung | Agent performance tracking and recognition |
| Aufmerksamkeit | Focus allocation based on energy and priority |
| Entwicklung | Quantum evolution layer — continuous growth |
| Sinn | Goals validated against meaning metric |

## Memory Architecture

### Obsidian Vault (External, Human-Readable)

Location: `~/Dokumente/Obsidian Vault/Orbit/`

Structure unchanged:
- Bewusstsein/ (daily consciousness logs)
- Reflexionen/ (agent reflections)
- Patterns/ (detected patterns)
- Stimmung/ (mood history)
- Agenten/ (agent births)
- Ziele/ (goal tracking)

The `qo-memory` crate reads and writes Markdown files via filesystem operations. No Obsidian API needed.

### HNSW Vector Store (Embedded)

Replaces Cognee (Port 8100) and Graphiti.

- Library: `instant-distance` (Rust, no external server)
- Dimensions: 768 (matching embedding model)
- Indices: agent-embeddings, pattern-embeddings, goal-embeddings, reflexion-embeddings, conversation-embeddings
- Search: <1ms for 100k vectors
- Persistence: memory-mapped file in data directory

### Embedding Generation

Text → Vector conversion uses a lightweight embedding model (e.g., `all-MiniLM-L6-v2` via `rust-bert` or ONNX runtime, 22 MB). This runs in-process alongside the IGQK model. The Tier 1 LLM is a generative model and not suitable for embeddings — a dedicated embedding model is needed.

### Persistent Storage

QO replaces SQLite with `redb` (embedded Rust key-value store, zero-config):
- Agent state, goal history, proposals, consciousness logs
- Single file: `data/qo.redb`
- No external process, no SQL, memory-mapped

## Tiered LLM Routing

A QLANG graph determines which tier handles each request based on complexity scoring.

| Tier | Model | Latency | Cost | Use Cases |
|------|-------|---------|------|-----------|
| 1 | QO-LLM (IGQK, ~31 MB) | <10ms | $0 | Routing, scoring, classification, simple analysis |
| 2 | Groq llama-3.3-70B | ~200ms | $0 (free tier) | Chat, reflection, research, strategy |
| 3 | Claude / DeepSeek | 1-5s | $$ | Complex architecture, code generation, deep reasoning |

### Complexity Router

```
graph llm_router {
  input prompt: String
  input task_type: Enum

  node complexity = score_complexity(prompt, task_type)
  node tier = cond(
    complexity < 0.3,  tier_1_local,
    complexity < 0.7,  tier_2_groq,
    tier_3_cloud
  )

  output response = tier
}
```

### Local Inference (Tier 1)

The Orbit fine-tuned Qwen2.5-0.5B is compressed locally via IGQK:
- Original: ~500 MB (f32 weights)
- Compressed: ~31 MB (ternary: {-1, 0, +1})
- Runs in-process in the Rust binary
- No Ollama, no external server

## Data Flow

A request through QO:

1. **User** types "Recherchiere X" → HTTP POST /api/chat
2. **qo-server** parses request, extracts intent
3. **qo-values** checks against 5 core values (deterministic, µs)
4. **qo-agents** runs CEO graph: validate → memory lookup → llm_reason → assign
5. **qo-llm** routes: "research" = Tier 2 (Groq 70B)
6. **qo-memory** stores result: HNSW embedding + Obsidian markdown
7. **qo-consciousness** updates state: mood=focused, energy -5%, pattern logged
8. **React Frontend** receives result + live consciousness via WebSocket

## Error Handling

Agent graphs handle errors at the graph level:
- Each node can return `Result<T, QoError>`
- On error, the graph follows a predefined error edge (not LLM-decided)
- Errors are typed: `LlmTimeout`, `MemoryNotFound`, `ValidationFailed`, `ValueRejected`
- The consciousness layer logs errors and adjusts mood (e.g., → Restless on repeated failures)
- No retries by default — the graph either succeeds or reports failure. Retry logic is explicit via `scan` with bounded iterations.

## Migration Phases

### Phase 1: Foundation

Extend Cargo workspace. Build qo-server (Axum), qo-llm (tiered router), qo-memory (HNSW + Obsidian bridge). Copy React frontend and adapt to new API. Result: QO starts, chat works via Groq.

### Phase 2: Agents

Build qo-agents crate. Implement 6 agent graphs as QLANG DAGs. Integrate LLM nodes. CEO can decompose goals and delegate to other agents. QLMS binary protocol for agent-to-agent communication.

### Phase 3: Consciousness

Build qo-consciousness + qo-values. State machine (heartbeat, mood, energy). Value scoring as deterministic graph node. SSE stream to frontend. Obsidian bridge for consciousness logs.

### Phase 4: Evolution

Quantum evolution layer. IGQK density matrix for pattern recognition. Compress Orbit-LLM locally with IGQK (Tier 1). Autonomous proposals based on quantum measurement.

### Phase 5: Data Migration

Import Orbit data: 258 generations, 1960 proposals, patterns, agent memories. Obsidian Vault stays in place (no move needed). Shut down Orbit, QO takes over.
